# Warm Kitty — 设计文档

> 日期：2026-05-31

## 一句话

一个 macOS 桌面 app：点一下"开始取暖"，它在后台跑满 CPU 让笔记本升温（给手/腿取暖），前台是一只**怕冷的小猫**，随 CPU 真实温度上升从"瑟瑟发抖"变到"满足"，到设定温度自动停止。

灵感来自 [Warm up your MacBook](https://z3ugma.github.io/2019/11/18/warm-up-your-macbook/)，把"跑满 CPU 发热"这个无聊动作套上一个好看、好玩、会心一笑的外壳。

## 目标与非目标

**目标**
- 简单：核心交互就是一个开始/停止按钮 + 一个目标温度滑块。
- 美观：主窗口，SwiftUI，一只有表情的小猫 + 真实温度数字。
- 有趣：小猫随温度变化的表情/动作 + 一句随温度变的旁白。
- 诚实安全：读真实温度；到目标温度自动停；不会烧坏机器。

**非目标（v1 不做）**
- 不上 Mac App Store。
- 不做菜单栏常驻。
- 不做强度/核心数等高级设置。
- 不做拟人之外的多角色/换肤。

## 平台与分发

- 平台：macOS（Apple Silicon 为主），SwiftUI，原生 Swift。
- 分发：**Developer ID 签名 + 公证**，自己网站直接下载（同 tutti 路线）。
- 不走 App Store：沙盒禁止随意跑满 CPU、禁止读温度传感器，且这类 app 大概率被人工审核拒绝。Developer ID 公证只做自动恶意软件扫描，不审内容，可顺利通过。

## 核心交互（v1）

1. 打开 app → 一只缩成一团、发抖的小猫 + 当前温度。
2. 一个大的"开始取暖"按钮 + 一个目标温度滑块。
   - 滑块范围 **60–95°C（CPU die 温度）**，**默认 80°C**。
   - 默认值的意义：小白不动滑块直接点开始也合理——80°C 是机身明显发热、又安全低于约 100°C 热节流线的水平。
3. 点开始 → 后台加热引擎启动，CPU 升温 → 小猫随温度变精神。
4. 到目标温度 → 自动停止，小猫进入"暖和够了"的满足态。
5. 任何时候可手动停止；关窗/退出必停。

## 模块拆分（各自单一职责，可独立测试）

### 1. ThermalReader（读真实温度）
- 用 IOHIDEventSystem 匹配热传感器，约 1s 轮询，对外暴露"当前温度"发布流（Combine `@Published` 或 `AsyncStream`）。
- 私有 API 全部封装在这一个文件里；读不到时返回 `nil`。
- 接口：`var currentTemperature: Double? { get }`（响应式）。

### 2. HeatingEngine（加热引擎）
- 纯 Swift 起 N 个 worker 线程跑紧浮点循环产生 CPU 负载。
- **不调用 `yes` / `stress` 外部命令**（无外部依赖、好控制、无子进程沙盒问题）。
- v1 用全核心、固定强度；强度预留占空比（工作/休息比例）参数，v1 不暴露给用户。
- 接口：`start()` / `stop()`。

### 3. PetState（状态机）
- 把温度档位映射到小猫状态：`发抖 → 苏醒 → 舒展 → 满足 → 兴奋 → 过热警告`。
- 同时产出当前旁白文案（承接"打比方"创意，如"现在像一杯热可可"）。
- 纯函数式映射，输入温度（和降级时的"已加热时长"），输出状态 + 文案，易测。

### 4. SessionController（编排）
- 开始/停止；监听 ThermalReader，达到目标温度即停 HeatingEngine。
- 兜底：最长加热时长上限（防温度读不到时无限跑）。
- 持有目标温度设置。

### 5. 主视图（SwiftUI）
- 小猫动画 + 温度数字 + 大开始/停止按钮 + 目标温度滑块。

## 数据流（闭环）

```
HeatingEngine 跑 CPU
   ↓ 机器真实升温
ThermalReader 读到温度上升
   ↓
PetState 切换状态 + 旁白
   ↓
小猫动画/文案变化
   ↓ 到目标温度
SessionController 停掉 HeatingEngine
   ↓
小猫进入"满足"态
```

## 自动停止与安全

- 主停止条件：达到设定目标温度。
- 兜底：最长加热时长上限；关窗 / 退出 App 必停。
- macOS 自带热节流（约 100°C），不会烧坏机器。
- 文案轻提示："取暖时会耗电、风扇会响。"

## 降级策略

私有热传感器 API 在某些机型或未来 macOS 大版本失效时：
- 温度数字位显示"--"。
- 小猫状态改由"已加热时长 + 强度"驱动。
- 加热功能照常工作。

→ 保证 app 永远不会变废。

## 留给视觉设计（Claude design）的方向

不锁死视觉，定方向：

- 角色：一只**怕冷的小猫**。
- 温度档 → 表情/动作映射：
  - 低温：缩成一团、瑟瑟发抖 / 睡着
  - 升温：慢慢睁眼、舒展身体
  - 适温：满足、打呼噜、冒幸福气泡
  - 高温：兴奋、冒汗但开心
  - 过热：夸张扇风 / 求饶表情 + 警告色
- 一句随温度变化的旁白，承接"打比方"创意。

## 待定 / 风险

- 私有热传感器 API 未来 macOS 大版本可能需适配——已知、可接受，有降级兜底。
- 后续可选项（非 v1）：菜单栏状态、强度设置、换猫皮肤。
